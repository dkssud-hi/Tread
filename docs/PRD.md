# PRD — Tread (트레드)

## 1. 서비스 개요
- **서비스 이름**: Tread (트레드)
- **한 줄 설명**: 내 기호에 맞는 러닝화를 찾고 싶을 때 쉽게 찾을 수 있는 서비스
- **핵심 가치**: 훈련의 목적, 착화감, 훈련 시 주행거리 등 조건들에 부합하는 러닝화를 쉽게 찾을 수 있다.

## 2. 문제 정의
나는 러닝을 즐겨하는 사람인데, 점점 러닝에 관심이 많아지면서 lsd, 인터벌 훈련, 조깅 각각의 목적에 맞는 러닝화를 찾게 되더라고. 근데 내가 원하는 조건에 맞게 일일히 구글링 해서 정보를 서칭하기 힘들어.

## 3. 타겟 사용자
- **주요 사용자**: 20대 이상의 러닝을 즐겨하는 남,녀
- **사용자의 니즈**: 본인의 훈련 목적, 착화감, 훈련 시 주행거리 등을 입력하면 조건에 부합하는 러닝화들을 추천해준다.

## 4. 핵심 기능

### Must Have
- 조건을 선택하여 검색하면 조건에 부합하는 러닝화를 나열해준다.
- 검색된 러닝화를 클릭하면 러닝화 상세 페이지에서 해당 러닝화의 특성을 설명해준다.

### Nice to Have
- 상세 페이지에서 구매하기 클릭하면 공식 홈페이지의 해당 모델 구매 페이지로 리다이렉트된다.
- 전국 마라톤 대회 캘린더

## 5. 사용자 시나리오

### 시나리오 1 — 입문자가 첫 러닝화를 고를 때
20대 직장인 A는 최근 러닝을 시작했지만 어떤 신발이 좋은지 몰라 막막하다. Tread에 접속해 훈련 목적은 '조깅', 주행거리(훈련 1회당 평균)는 '5km 이하', 착화감은 '푹신한 쿠셔닝'을 선택한다. 조건에 맞는 입문용 데일리 트레이너 목록이 나오고, 그중 하나의 상세 페이지에서 쿠셔닝·무게·드롭 등 특성을 확인한 뒤 '구매하기'를 눌러 브랜드 공식몰로 이동해 구매한다.

### 시나리오 2 — 목적별로 신발을 나눠 쓰려는 중급 러너
주 4~5회 달리는 B는 훈련 종류별로 신발을 구분하고 싶다. 평소엔 데일리화를 신지만, 인터벌과 템포 훈련에는 반발력 좋은 신발이 필요하다고 느낀다. '인터벌/스피드' 목적과 '가벼움', '반발력 중시'를 선택해 카본 플레이트가 들어간 모델들을 추천받고, 여러 모델의 상세 스펙을 비교한 뒤 마음에 드는 모델을 골라 구매 페이지로 이동한다.

### 시나리오 3 — 대회를 앞두고 레이스화를 찾는 사용자
하프 마라톤을 준비 중인 C는 '대회/레이스' 목적과 '장거리(20km 이상)', '경량'을 조건으로 검색한다. 추천된 레이스화의 상세 페이지에서 무게와 반발 특성을 확인하고, (Nice to Have가 구현된 경우) 마라톤 캘린더에서 참가 예정 대회 일정을 함께 확인한 뒤 구매를 결정한다.

## 6. 성공 지표

### 핵심 지표 (North Star)
- **Must Have 기능 정상동작**: 조건 검색 → 결과 나열, 상세 페이지 특성 설명이 정상적으로 동작한다.

> 초기 목표 수치는 베타 운영 데이터를 확보한 뒤 기준선(baseline)을 설정하여 정의한다.

## 7. 기술 요구사항

- **성능**:
  - 검색 결과는 사용자가 조건을 적용한 뒤 2초 이내 표시되는 것을 목표로 한다.
  - 상세 페이지 초기 로딩 1.5초 이내.
  - 동시 접속 증가에 대비해 검색 API는 캐싱 및 인덱싱을 적용한다.

- **보안**:
  - 모든 통신은 HTTPS로 암호화한다.
  - 회원 기능 도입 시 개인정보(이메일 등)는 암호화 저장하고 개인정보보호법을 준수한다.
  - 외부 공식몰 리다이렉트 시 신뢰할 수 있는 도메인만 허용(URL 검증)한다.

- **플랫폼 전략 (단계적 출시)**:
  - **1단계 — 웹 우선 (MVP)**: Next.js 기반 반응형 웹 + PWA로 먼저 출시하여 빠르고 저비용으로 시장 반응을 검증한다. 별도 설치 없이 브라우저 URL로 접근 가능하며, PWA로 모바일 홈 화면 추가도 지원한다.
  - **2단계 — 모바일 앱**: 웹 검증 후 iOS / Android 앱을 추가한다. 앱은 기존 백엔드 API를 그대로 재사용한다.
  - 웹과 앱은 동일한 백엔드 API와 데이터를 공유한다.
  - 검색 결과·러닝화 상세 페이지는 각각 고유 URL을 가져 공유 및 검색엔진(구글 등)을 통한 유입이 가능하다.

- **기술 스택 / 환경** (제안):
  - **웹 (1단계)**: Next.js 기반 반응형 웹 + PWA. 페이지마다 고유 URL을 부여하고, 검색엔진 노출을 위해 SSR(서버 사이드 렌더링)을 적용한다.
  - **모바일 앱 (2단계)**: React Native / Flutter(크로스 플랫폼) 또는 iOS·Android 네이티브
  - **백엔드**: Node.js / Python 기반 REST API (웹·앱 공통)
  - **데이터베이스**: 러닝화 정보 저장용 RDB(PostgreSQL/MySQL) + 검색 성능 향상을 위한 검색 인덱스(선택)
  - **배포/호스팅**: 클라우드(AWS/GCP/Vercel 등), 앱은 추후 App Store / Google Play 배포
  - **데이터 소스**: 브랜드 공식 정보 기반 러닝화 스펙 DB(목적, 쿠셔닝, 무게, 드롭, 추천 주행거리, 가격, 공식 구매 링크 등)

## 8. 범위 외 (Out of Scope)
- 자체 결제 / 직접 판매 (구매는 공식몰 리다이렉트로 처리, 인앱 결제·장바구니 없음)
- 재고·가격 실시간 연동 및 정산
- 사용자 리뷰·평점·커뮤니티 기능
- 개인 맞춤 발 사이즈 측정 / AI 추천 고도화 (초기 버전은 조건 기반 필터링까지만)
- 중고 거래 및 리셀 기능
- 의류·양말 등 러닝화 외 용품 추천
